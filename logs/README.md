# Logs

## Overview

This directory contains experiment logs generated using the **SET scheduling framework**.
All experiments are conducted on **Transformer neural network workloads**.

The experiments investigate how different **task granularities of Transformer operators** interact with different **inter-layer scheduling strategies** on tiled accelerators.

Each log records:

* the scheduling tree structure
* the scheduling scheme
* performance statistics such as energy and latency

## 概述

本目录存储使用 **SET 调度框架**运行实验产生的日志。
所有实验均基于 **Transformer 神经网络模型**。

这些实验主要研究 **Transformer 算子任务粒度** 与 **不同层间调度策略** 在 tiled accelerator 上的相互影响。

每个日志记录：

* 调度树结构
* 调度方案
* 性能指标（能耗、延迟等）

---

## Framework

All experiments are built upon the SET framework proposed in the ISCA 2023 paper:

**SET: Inter-layer Scheduling Space Definition and Exploration for Tiled Accelerators**

SET represents scheduling structures using a **Resource Allocation Tree (RA Tree)** and automatically explores the inter-layer scheduling space.

## 框架

所有实验均基于 ISCA 2023 论文提出的 **SET 调度框架**：

**SET: Inter-layer Scheduling Space Definition and Exploration for Tiled Accelerators**

SET 使用 **资源分配树（RA Tree）** 表示调度结构，并自动探索层间调度空间。

# Granularity Types

The experiments evaluate several task granularities derived from **Transformer workloads**.

## 任务粒度类型

实验评估了多种基于 **Transformer 网络结构**设计的任务粒度。

---

## base

The original Transformer network structure where each operator is scheduled independently.

Each Transformer operator (e.g., attention, feed-forward, element-wise operations) is treated as an independent scheduling unit.

## base（原始粒度）

原始 Transformer 网络结构，每个算子作为独立任务进行调度。

Transformer 中的 attention、feed-forward、element-wise 等算子均作为独立调度单元。

---

## semigrouped

Semi-grouped granularity for Transformer attention structures.

The attention-related operators inside a Transformer block are **divided into two groups**, forming two scheduling units.

This results in a slightly coarser granularity than the base configuration while preserving the original network structure.

## semigrouped（双小包粒度）

针对 Transformer attention 结构设计的 **半打包粒度**。

Transformer block 中的 attention 相关算子 **被划分为两个小组**，形成两个调度单元。

相比 base，该策略略微增大任务粒度，但 **不改变网络结构**。

---

## grouped

Grouped granularity.

All attention-related operators inside a Transformer block are **combined into a single scheduling group**.

This results in a larger scheduling unit compared with `semigrouped`.

## grouped（单次打包粒度）

打包粒度。

Transformer block 中 **所有 attention 相关算子被合并为一个调度单元**。

相比 `semigrouped`，该策略形成 **更大的任务粒度**。

---

## fused

Layer fusion granularity.

Multiple consecutive layers are **merged into a single fused layer** before scheduling.

Unlike grouping, this transformation **modifies the neural network structure**.

## fused（层融合粒度）

层融合粒度。

多个连续神经网络层在调度前 **融合为一个新的层**。

与 grouped 不同，这种方式 **会改变神经网络结构**。

---

## Granularity Relationship

The granularity relationship of the evaluated configurations is:

```
base → semigrouped → grouped → fused
(finer)                     (coarser)
```

## 粒度关系

实验中不同粒度策略的关系如下：

```
base → semigrouped → grouped → fused
（最细）                     （最粗）
```

# Scheduling Methods

Different inter-layer scheduling strategies are evaluated.

## 调度方法

实验评估了多种层间调度策略。

---

## init

Initial scheduling configuration before optimization.

## init

优化前的初始调度结构。

---

## LS

Layer Sequential scheduling.

Layers are executed sequentially and all hardware resources are allocated to one layer at a time.

## LS

层顺序执行（Layer Sequential）。

一次只执行一层，所有硬件资源集中用于当前层。

---

## LP

Layer Pipeline scheduling.

Different layers execute concurrently in a pipeline across hardware partitions.

## LP

层流水执行（Layer Pipeline）。

不同层在不同硬件分区上并行执行，形成流水线。

---

## SET

Scheduling result generated by the **SET scheduling exploration algorithm**.

SET automatically searches the inter-layer scheduling space using RA Trees.

## SET

由 **SET 调度空间探索算法**生成的调度结果。

SET 使用 RA Tree 表示调度结构并自动搜索层间调度空间。

# Log File Types

Each experiment generates three types of logs.

## 日志类型

每个实验会生成三类日志文件。

---

## tree file

Example:

```
base_SET_tree.txt
```

This file records the **Resource Allocation Tree (RA Tree)** representing the scheduling structure.

## tree 文件

示例：

```
base_SET_tree.txt
```

该文件记录 **资源分配树（RA Tree）**，表示调度结构。

---

## scheme file

Example:

```
base_SET_scheme.txt
```

This file contains the detailed scheduling scheme including task allocation and execution configuration.

## scheme 文件

示例：

```
base_SET_scheme.txt
```

该文件记录详细调度方案，包括任务分配与执行配置。

---

## summary file

Example:

```
base_SET_summary.txt
```

This file reports the overall performance statistics of the experiment.

Typical metrics include:

* Energy
* Latency
* Cost Function

Example:

```
Energy: 5.02481e+11
Latency: 257634112
Cost Function: 1.29456e+20
```

## summary 文件

示例：

```
base_SET_summary.txt
```

该文件记录实验整体性能指标。

常见指标包括：

* 能耗（Energy）
* 延迟（Latency）
* Cost Function

示例：

```
Energy: 5.02481e+11
Latency: 257634112
Cost Function: 1.29456e+20
```
